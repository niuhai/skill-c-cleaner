# C盘智能决策顾问 v6.0 (AI Decision Layer)

## 🎯 一句话定位

**我不是清理工具，我是你的 AI 磁盘健康顾问。**

```
传统工具（CCleaner/BleachBit/WizTree）= 执行层：帮你删文件（但不懂你）
本 Skill（AI Decision Layer）          = 决策层：理解你 + 分析数据 + 智能建议 + 教你安全执行
```

**核心价值**: 不是"帮你打扫"，而是 **"教你如何管理系统空间 + 根据你的情况做最优决策"**。

---

## 🧠 为什么需要 AI 决策层？

### 传统工具的困境

| 场景 | 传统工具的做法 | 问题 |
|------|---------------|------|
| 发现 10GB node_modules | 显示"可删除" | 用户不敢删：万一以后要用呢？ |
| C盘只剩 15GB | 列出所有可删项 | 用户焦虑：先删哪个？会不会出问题？ |
| 开发者 vs 普通用户 | 同样的清理列表 | 没有个性化：开发者的缓存可能很重要 |
| 发现可疑大文件 | 只显示大小 | 不告诉你这是什么、能不能删 |

### AI 决策层的解决方案

| 场景 | AI 的做法 | 价值 |
|------|----------|------|
| 发现 10GB node_modules | 分析项目活跃度 → 判断可删除 → 告诉你 `npm install` 可恢复 | ✅ 放心决策 |
| C盘只剩 15GB | 生成分阶段紧急方案 → 按风险排序 → 预期效果量化 | ✅ 有条不紊 |
| 开发者 vs 普通用户 | 识别用户类型 → 定制策略 → 平衡"清洁"与"效率" | ✅ 个性化 |
| 发现可疑大文件 | 识别文件用途 → 风险评估 → 给出处理选项 | ✅ 知其所以然 |

### 三大核心优势

#### 1️⃣ **上下文感知（Context-Aware）**
```
用户："我刚装了 Docker"
AI："检测到 Docker 占用 15GB（镜像 12GB + 数据卷 3GB）。
     建议：如果只是测试用，可以清理未使用的镜像（命令...）；
     如果长期使用，建议迁移到 D 盘（附完整迁移教程）。
     另外，我注意到你的 WSL 也占用了 8GB，是否一并处理？"
```
**价值**: 不是机械地列出文件，而是结合你的操作历史给出 timely 的建议。

#### 2️⃣ **动态风险评估（Dynamic Risk Assessment）**
```
用户："这个 NuGet 缓存 323MB 能删吗？"
AI："可以删，但要注意：
     ✅ 安全性：NuGet 包可以从网络重新下载
     ⚠️ 但我发现你上周编译过 ProjectX（使用 NuGet 依赖），
        如果你近期还要改代码，删除会导致下次编译变慢。
     💡 建议：如果你本周不再开发 .NET 项目，可以安全删除；
        否则建议保留，或者设置环境变量将 NuGet 缓存指向 D 盘。"
```
**价值**: 不是简单的"能/不能删"，而是基于你的实际情况给出一分为二的分析。

#### 3️⃣ **教育型输出（Educational Output）**
```
不是："运行 clean-safe.ps1"
而是："为什么要清理临时文件？
      因为 Windows 和应用程序会产生大量临时文件，
      正常情况下程序退出时会自动删除，但异常退出的会残留。
      这些文件累积起来可能占用数 GB 空间，
      而且百分之百可以安全删除（程序会在需要时重建）。
      
      执行命令：.\cleaners\clean-safe.ps1 -WhatIf （先预览）
                .\cleaners\clean-safe.ps1 -ReallyDelete （确认后执行）"
```
**价值**: 让用户理解"为什么"，而不只是"怎么做"。建立信任，赋能用户。

## 🎨 产品哲学

### 设计原则

#### 1. **只读优先（Read-First）**
- ✅ 所有扫描脚本都是只读的
- ✅ 不自动删除任何文件
- ✅ 输出的是 **带决策的建议报告**，而不是执行结果
- **理由**: 信任是逐步建立的。让用户看到 AI 的分析质量，再决定是否采纳。

#### 2. **教育赋能（Empowerment Over Automation）**
- ❌ 不是"我帮你删了"
- ✅ 而是"我分析了，建议这样做，理由是...，你要不要执行？"
- **目标**: 让用户学会自己管理空间，AI 是顾问而非保姆。

#### 3. **个性化适配（Personalization）**
- 识别用户类型：开发者 / 设计师 / 普通用户 / 企业环境
- 根据使用习惯调整建议策略
- 动态计算风险/收益比
- **拒绝一刀切**。

#### 4. **透明可解释（Explainable AI）**
- 每个建议都有明确的理由
- 展示决策过程（不仅仅是结论）
- 承认不确定性（"我不确定 X，建议你手动检查"）
- **建立信任的关键**。

#### 5. **渐进式信任（Progressive Trust Building）**
- 首次使用：只读分析 + 建议
- 多次使用后：可以授权自动化（如定时任务）
- 始终保留人工确认环节
- **尊重用户的掌控权**。

## 🔬 工作流程

### 完整的用户旅程

```
1️⃣ 用户触发
   ↓ "我的C盘快满了" / "帮我分析一下C盘"

7️⃣ 用户选择执行项
   ↓ AI 提供精确的执行命令 + 安全提示

8️⃣ （可选）用户执行 + 反馈
   ↓ AI 记录偏好，未来建议更精准
```

## 🔧 扩展性设计

### 数据与代码分离

```
扫描脚本(只读)  ──读取──→  app-signatures.json (100+已知应用, 14类别)
                         user-custom.json   (用户自己的软件)

scan-discover.ps1 ──发现未知大文件夹──→ 自动生成扩展建议
                                    └──→ 用户确认后合并到 user-custom.json
```

**核心理念**: 新软件签名只需一行 JSON，无需修改 PowerShell 代码。

### 如何添加自定义软件

编辑 `extensions/user-custom.json`：

```json
{
  "name": "我的特殊软件",
  "detect_paths": ["%LOCALAPPDATA%\\MyApp\\cache"],
  "cleanable": true,
  "note": "这是我装的软件，缓存可清理",
  "risk_level": "safe",  // safe / warning / danger
  "custom_advice": "删除后会重新下载，约需 5 分钟"
}
```

保存后，下次扫描自动生效。

### 流氓软件自动识别

`app-signatures.json` 中 `rogue_software` 类别收录了 17 种常见国产流氓/捆绑软件。

扫描时会自动标记 ⚠️ 并给出卸载建议。

## 🛡️ 安全原则

### 风险等级体系

| 等级 | 标记 | 含义 | 处理方式 |
|------|------|------|---------|
| ✅ | **强烈推荐** | 纯临时文件、缓存，删除无副作用 | 可直接执行 |
| ⚠️ | **确认后操作** | 需要用户人工确认的低风险项 | 建议备份后执行 |
| ❌ | **不建议/禁止** | 系统核心、企业管控、用户数据 | 绝不触碰 |
| 🔴 | **必须人工检查** | 可疑文件、未知大文件 | 引导用户自行判断 |

### 安全红线（永远不自动执行）

- ❌ 删除系统还原点
- ❌ 关闭休眠 / 移动页面文件
- ❌ 清理 WinSxS `/ResetBase`
- ❌ 删除 Program Files 下任何目录
- ❌ 删除用户文档/桌面/下载
- ❌ 触碰企业安全软件(EDR/NAC/杀毒引擎文件)
- ❌ vssadmin / diskpart 操作

### 安全机制

1. **只读默认**: 所有扫描脚本只读不写
2. **WhatIf 模式**: 清理脚本支持 `-WhatIf` 预览
3. **明确确认**: 危险操作需要 `-ReallyDelete` 参数
4. **快照备份**: 提供 `safety/snapshot-before-after.ps1`
5. **回滚指南**: 提供 `safety/rollback-guide.md`

3. 提供分步命令，按风险排序
4. 预期效果：30分钟后 C盘可用空间翻倍

### 场景 3: 特定应用咨询

**用户**: "Docker 太占空间了"

**AI 执行**:
1. 扫描 G 类（特殊占用源）
2. 分析 Docker 占用详情：
   - 镜像: 8GB（列出未使用的）
   - 数据卷: 3GB
   - WSL2 发行版: 5GB
3. 提供选项：
   - A) 清理未使用镜像（低风险，释放 3GB）
   - B) 迁移到 D 盘（中风险，释放 16GB）
   - C) 完全重置（高风险，释放全部）
4. 附完整迁移教程和命令

### 场景 4: 学习最佳实践

**用户**: "怎么避免 C 盘再次被占满？"

**AI 执行**:
1. 分析历史增长趋势
2. 识别主要增长源（如：npm 缓存每月增长 2GB）
3. 生成长期预防方案：
   - 安装路径规范（新软件装 D 盘）
   - 开发环境变量配置（缓存指向 D 盘）
   - 定期自动清理任务设置
   - 浏览器缓存大小限制
4. 提供自动化脚本

## 🤝 与开源工具的关系

### 我们不是替代品，是互补品

```
┌─────────────────────────────────────────────┐
│              用户的工作流程                   │
│                                             │
│  1️⃣ c-drive-cleaner (AI)                   │
│     ↓                                       │
│     深度分析 + 智能决策 + 生成方案           │
│     ↓                                       │
│  2️⃣ 用户确认方案                            │
│     ↓                                       │
│  3️⃣ BleachBit / WizTree / BCU              │
│     ↓                                       │
│     根据 AI 方案执行具体删除操作             │
│                                             │
└─────────────────────────────────────────────┘
```

**比喻**:
- **AI (本 Skill)** = 建筑师（设计蓝图、规划方案）
- **BleachBit/WizTree** = 施工队（按蓝图执行）

两者结合，效果最佳。

### 推荐工具组合

| 任务 | 推荐工具 | 理由 |
|------|---------|------|
| **初步分析** | **本 Skill (AI)** | 全面 + 智能 + 个性化 |
| **可视化大文件** | WizTree | 极快的 TreeMap 可视化 |
| **批量删除** | BleachBit | 成熟的清理规则库 |
| **程序卸载** | BCU (Bulk Crap Uninstaller) | 彻底卸载残留清理 |
| **手动检查** | Explorer + TreeSizeFree | 精细控制 |
